# Nebula North Star

Nebula must become a production-grade, real-time 3D mind-map for project/file trees that is reliable under scale, visually clear, and fast to operate. Every automated change must improve at least one of these pillars without degrading the others: **reliability** (stable builds, deterministic behavior, safe rollback), **performance** (render/runtime efficiency, bounded resource use), **usability** (clear graph navigation and operator control), and **maintainability** (typed contracts, testable modules, auditable operations). The hourly RALPH loop should only ship bounded, reversible patches that move the system toward this end-state and leave a traceable validation trail. Use `docs/upgrades/VISUAL_REFERENCE.md` as inspiration context for visual direction while prioritizing measurable performance and reliability.
