# Nebula UI + Server Audit

**Timestamp:** 2026-03-10 01:52:44 EDT  
**Branch:** `agent`  
**Scope:** React-Flow runtime UX, source selection/loading reliability, server startup/runtime robustness.

## 1) Audit Findings

### A. UI pain points

1. **No source-selection path in active UI (blocking start)**
   - **Symptom:** App can load with empty graph and no obvious way to select source when server requires explicit source selection.
   - **Repro:** Start server with default `UNIVERSE_REQUIRE_SOURCE=true` and no preselected root. Open `/`.
   - **Likely root cause:** Active entry (`UniverseFlowScene`) did not include source modal/selector flow.

2. **Weak loading/connection state visibility**
   - **Symptom:** Users see disconnected/empty UI but not clear whether it is source missing, WS bootstrap delay, or server unreachable.
   - **Root cause:** WS provider only toggled boolean connected state; no phases, retries, or user-facing message.

3. **Viewport/panel crowding on smaller viewports**
   - **Symptom:** Panel and top HUD overlap at smaller widths/heights; tree viewport can feel cramped.
   - **Root cause:** Panel anchored `top-3` with fixed width and static `62vh` scroller.

4. **Source modal flow feedback gaps**
   - **Symptom:** Source API and selection failures surface as raw errors with limited timeout behavior.
   - **Root cause:** API client had no fetch timeout and unguarded JSON assumptions.

5. **Focus/radial mode constraint check**
   - **Result:** Existing radial focus behavior is active in `ReactFlowOverlay` and was preserved intentionally.

### B. Server loading/startup/runtime pain points

1. **Default-root autoselect could hard-fail startup path**
   - **Symptom:** If `UNIVERSE_AUTOSELECT_DEFAULT_ROOT=true` and default root invalid/forbidden, bootstrap could fail hard.
   - **Root cause:** `bootstrap()` awaited `startRoot(DEFAULT_ROOT)` without resilience fallback.

2. **API env defaults less robust than WS side**
   - **Symptom:** API port had narrower env compatibility than expected in some deployment setups.
   - **Root cause:** API port parsing only looked at `UNIVERSE_API_PORT`.

3. **WS bootstrap ambiguity from frontend perspective**
   - **Symptom:** WS retries happen, but user-facing app gave little guidance.
   - **Root cause:** frontend connection layer didn’t expose retry/error phases.

## 2) Fixes Implemented

### UI + frontend runtime fixes

1. **Restored and integrated source-selection flow into active scene** (`UniverseFlowScene`)
   - Added source modal with:
     - allowed-root list
     - folder list by root
     - manual absolute-path input
     - submit with in-modal loading + errors
   - On first load:
     - checks `/source/current`
     - opens modal when no current root
     - keeps WS disabled until source selected

2. **Connection lifecycle UX improvements** (`UniverseLiveProvider`, `wsClient`)
   - Added explicit connection phases: `idle | connecting | connected | retrying | error`
   - Surfaced retry cadence and WS error messages through `onStatus`
   - Status HUD now displays phase and concise message

3. **Loading overlays and startup state clarity** (`UniverseFlowScene`)
   - Added deterministic loading overlays for:
     - source-configuration check
     - runtime connect phase

4. **Panel viewport/readability adjustments** (`ProjectViewerPanel`)
   - Responsive panel width (`min(360px, viewport)` style)
   - Moved panel down (`top-16`) to reduce HUD overlap
   - Dynamic list height (`100dvh`-based) for better small-screen behavior

5. **Source API reliability + timeout behavior** (`sourceApi.ts`)
   - Added request timeout (9s) with explicit timeout error
   - Defensive JSON parse handling and payload validation
   - More stable error propagation for source endpoints

### Server/runtime robustness fixes

6. **Startup hardening for default autoselect** (`apps/server/src/index.ts`)
   - Wrapped default-root autoselect in guarded try/catch
   - Logs warning + continues in selection mode (instead of failing whole startup)

7. **API env fallback improvement** (`apps/server/src/index.ts`)
   - Port resolution now accepts `UNIVERSE_API_PORT` and `API_PORT`
