## Findings

### 1. High: generated directories are back in the live graph at the same time the physics loop became quadratic

Recent committed code removed `dist`, `build`, `.next`, `.cache`, `.turbo`, `.parcel-cache`, and `coverage` from the watcher ignore list, so the server now ingests those trees instead of dropping them at source. See [apps/server/src/universe/ignore.ts](/Users/josh/.openclaw/workspace/projects/nebula/apps/server/src/universe/ignore.ts#L11) and [apps/server/src/universe/index.ts](/Users/josh/.openclaw/workspace/projects/nebula/apps/server/src/universe/index.ts#L18).

The current uncommitted worktree then adds `solveCollisionConstraints`, which does a nested pairwise pass over all nodes for each tick, plus anchor clamping on every step. See [packages/physics/src/verlet.ts](/Users/josh/.openclaw/workspace/projects/nebula/packages/physics/src/verlet.ts#L152) and [apps/server/src/universe/graph.ts](/Users/josh/.openclaw/workspace/projects/nebula/apps/server/src/universe/graph.ts#L241).

Impact:

- The client may hide these nodes by default, but the server still watches them, stores them, and simulates them.
- Large rebuilds can now inflate node count exactly where the new solver is most expensive.
- With `NEBULA_PHYSICS_TICK_MS` defaulting to 60 ms, this can turn background build output into a sustained CPU cost.

Recommendation:

- Either restore server-side ignores for generated trees, or gate collision solving behind a node-count threshold / broad-phase filter before shipping this worktree.

### 2. Medium: clicking a summary chip can collapse the branch instead of revealing more content

Summary nodes are rendered not only for collapsed folders, but also for expanded folders that exceed the child budget. In that expanded case, clicking the summary should reveal more of the folder, but the current handler always calls `toggleExpandedNode(parentId)`. See summary-node creation at [apps/web/src/universe/ReactFlowOverlay.tsx](/Users/josh/.openclaw/workspace/projects/nebula/apps/web/src/universe/ReactFlowOverlay.tsx#L341) and the click handler at [apps/web/src/universe/ReactFlowOverlay.tsx](/Users/josh/.openclaw/workspace/projects/nebula/apps/web/src/universe/ReactFlowOverlay.tsx#L569).

Impact:

- If a focused directory is already expanded and still budget-limited, clicking `+N hidden` collapses the directory.
- This makes the affordance behave opposite to the user’s likely intent.

Recommendation:

- Only toggle expansion when the parent is currently collapsed, or replace the toggle with a dedicated "increase budget / drill in" behavior for summary nodes.

### 3. Medium: the new hidden-node rule is broad enough to hide real content, not just bloat

The new visibility helper hides any non-root node whose path contains a segment named `docs`, `dist`, `build`, `.next`, `.cache`, `.turbo`, `.parcel-cache`, or `coverage`, plus generic `cache` suffixes. See [apps/web/src/universe/visibility.ts](/Users/josh/.openclaw/workspace/projects/nebula/apps/web/src/universe/visibility.ts#L3). The drawer and search both apply this filter before computing results. See [apps/web/src/universe/ProjectViewerPanel.tsx](/Users/josh/.openclaw/workspace/projects/nebula/apps/web/src/universe/ProjectViewerPanel.tsx#L47) and [apps/web/src/universe/ReactFlowOverlay.tsx](/Users/josh/.openclaw/workspace/projects/nebula/apps/web/src/universe/ReactFlowOverlay.tsx#L205).

Impact:

- Any legitimate subtree under `docs/` is hidden by default, even when it is part of the actual product surface.
- Search results also disappear until the user discovers and enables `show hidden`.
- This is a behavior change from server-side ignoring to client-side hiding; the data now exists but defaults to invisible.

Recommendation:

- Narrow the rule to build artifacts only, or make the label explicit that `docs` is included in the hidden set.
